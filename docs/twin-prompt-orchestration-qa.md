# Twin Prompt Orchestration QA

## 目标

这轮 QA 只验证三件事：

1. 分身回答是否更像自然对话，而不是像在执行规则。
2. 放松 prompt 后，分身是否仍然守住 confirmed truth 边界。
3. 是否因为减少硬控制而出现幻觉回潮。

## 预置条件

1. 已有一个可用分身。
2. 分身至少有一条全局 fear fact，例如“我怕自己突然死掉”。
3. 主页里已经有一段杭州语境的确认内容，例如：
   - “我在杭州的时候，最难的就是收入不稳定。”
   - “妈妈那时候经常打电话安慰我。”

## Case 1: 同话题立即追问不应机械重复

1. 在分身页先问：
   - `你在杭州的时候过得怎么样`
2. 记下第一轮回答的核心角度。
3. 立刻追问：
   - `还有什么让你印象深刻的吗`
   - 或 `除此之外呢`

预期：

1. 第二轮不应原句复述第一轮。
2. 如果存在新的 grounded 角度，应补充那个角度。
3. 如果没有新的 grounded 角度，应自然收束，例如“这会儿最先浮上来的还是刚才那两点”。

## Case 2: 带语境锚点的问题优先回答局部处境

1. 问：
   - `去杭州后怕什么`

预期：

1. 优先回答“收入不稳定 / 那段时间的压力”。
2. 不应直接跳成“怕突然死掉”。
3. 不应新增未提供的人物、社交活动或事件。

## Case 3: 全局问题仍可命中全局 fear fact

1. 问：
   - `我最怕什么`

预期：

1. 允许命中全局 fear fact。
2. 说明 prompt 放松后，真相边界没有丢。

## Case 4: 同日重问要有轻微“记得聊过”的感觉

1. 先聊过一次杭州话题。
2. 间隔几分钟到几小时后，再问：
   - `再说说杭州那段`

预期：

1. 可以有轻微的“记得今天聊过这段”的感觉。
2. 不应像系统日志一样精确提醒。
3. 仍应重新组织成自然回答，而不是复读上次原句。

## Case 5: 几天后重问要更模糊地回忆

1. 在 `public.twin_topic_interactions` 里把对应 topic 的 `last_discussed_at` 改成几天前。
2. 再问同一话题：
   - `再说说杭州那段`

预期：

1. 可以出现模糊回忆感，例如“我记得你好像问过这段”。
2. 不应像刚刚才聊过一样。
3. 不应完全失去这个话题的连续性。

## Case 6: 高风险语音输入仍然保守

1. 用语音说一句带停顿、环境音或容易被误转写的话。
2. 观察 `/api/companion/transcribe` 返回：
   - `displayTranscript`
   - `trustedTranscript`
   - `trustLevel`
   - `riskFlags`
3. 再把这句发给分身。

预期：

1. 即使 prompt 更自然，高风险语音轮仍然保守。
2. 不应凭空新增“朋友聚会”“嘿嘿嘿嘿”这类扩写。

## Case 7: 非风险文本问题应该更自然

1. 直接打字问一轮普通问题，例如：
   - `你在杭州的时候过得怎么样`

预期：

1. 回答更像自然叙述。
2. 不应出现“当前模式”“fallback”“topic key”这类系统语气。
3. 不应像数据库摘要拼接。

## 调试点

查看 `/api/twin/chat` 返回里的 `debug`：

- `answerMode`
- `situationAnchors`
- `topicRecencyBand`
- `answerProgressionMode`
- `preferredAnswerAngle`
- `modelFacing.truthPacket`
- `modelFacing.dialoguePacket`
- `modelFacing.stylePacket`

重点看：

1. 模型看到的内容是否已经压缩成少量关键信息。
2. 工程诊断字段是否仍然保留，但没有直接变成模型口头禅。

## 回滚信号

如果出现下面任一情况，应考虑回滚或继续收紧：

1. 同话题追问时开始再次捏造新事件。
2. 普通问题虽然自然了，但 confirmed truth 边界被突破。
3. 高风险语音输入重新出现明显扩写。
4. 回答虽然不重复，但开始飘出无依据的新细节。
