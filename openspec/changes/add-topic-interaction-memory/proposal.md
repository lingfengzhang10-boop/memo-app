## 为什么

当前分身已经开始具备“人生记忆”，但还不具备“互动记忆”。同一个用户在同一段时间里连续追问同一话题时，分身会机械重复上一轮已经说过的内容；而隔一段时间后再次问起时，分身也不会表现出“我记得你问过这段”的自然模糊感。这会削弱陪伴感，让分身更像检索器而不是正在和人继续聊的人。

现在需要补这一层，是因为我们已经完成了语境路由和长期语义底座，但对话层仍然缺少“谁问过、多久前问过、上次说到哪一层”的运行时记忆。如果现在不补，这类重复和不自然回忆会持续放大。

## 变更内容

- 为分身增加轻量的话题互动记忆层，记录某个话题最近被谁问过、最近一次怎么回答、最近一次回答的角度和时间。
- 调整分身回答编排：同一 asker 在短时间内追问同一话题时，优先补充、换角度或诚实收束，而不是重复上一轮内容。
- 为同一 asker 的不同时间间隔增加不同的回忆感：刚聊过、同日再问、隔几天再问，应表现出不同强度的“记得我们聊过”。
- 区分 same asker 与 different asker：不同 asker 询问同一人生事实时，不应继承上一位 asker 的对话痕迹。
- 为分身回答增加“互动记忆解释字段”，便于调试当前是首次回答、递进回答、模糊唤回还是全新回答。

## 功能 (Capabilities)

### 新增功能
- `topic-interaction-memory`: 让分身对同一话题具备轻量互动记忆，能区分同一 asker 的短期追问、隔时重问和不同 asker 的首次提问。

### 修改功能
- `twin`: 分身对话从“只消费人生记忆”扩展为“人生记忆 + 互动记忆”共同决定回答方式，避免同会话机械复述。

## 影响

- 受影响的代码：`app/api/twin/chat/route.ts`、分身前端会话状态、分身上下文组装与运行时路由逻辑。
- 受影响的数据：需要新增轻量互动记忆表或等价持久化结构，用于记录 topic key、asker key、最近讨论时间、最近回答角度等。
- 受影响的 API：`/api/twin/chat` 需要接收或生成互动记忆上下文，并输出更可解释的 debug 字段。
- 不改变的边界：
  - 不改变“一人一分身”边界。
  - 不改变 confirmed facts / events 的正式资产边界。
  - 不让互动记忆反向污染长期人生事实或 active twin truth。
