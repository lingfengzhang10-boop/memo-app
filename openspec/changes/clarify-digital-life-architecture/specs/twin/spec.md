## ADDED Requirements

### 需求:分身必须作为受治理资产的消费面
分身必须被定义为长期数字资产的消费与呈现形态之一，而不是可以直接自由消费任意原始记忆的独立系统。

#### 场景:分身生成回复
- **当** 分身准备基于长期记忆生成回复
- **那么** 系统必须以经过筛选的长期资产上下文作为输入
- **那么** 系统禁止把未受治理的原始 transcript 直接当作默认回复依据

### 需求:分身不是唯一产品输出形态
系统必须允许与分身并列的其他数字生命呈现面存在，例如人生地图、关系图和传记视图。

#### 场景:产品扩展新的呈现面
- **当** 产品新增非聊天的数字生命呈现面
- **那么** 分身规范必须允许这些呈现面与分身共用同一套长期资产基础
- **那么** 系统禁止把分身设定为唯一的长期资产消费方
