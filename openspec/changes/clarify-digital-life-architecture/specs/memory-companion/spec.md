## ADDED Requirements

### 需求:日常记忆链路必须服务于数字资产沉淀
系统必须将日常语音链路定义为长期数字资产的统一入口，而不只是 facts / events 的提取管道。

#### 场景:一次讲述进入长期资产流
- **当** 用户完成一次日常语音讲述并触发后台理解
- **那么** 系统必须将这次内容视为后续事实、事件、人物、地点、叙事或表达资产的潜在来源
- **那么** 系统禁止将 transcript 视为唯一长期记忆形态

### 需求:不同资产层必须保持分层边界
系统必须保持原始层、客观层、叙事层与表达层分离，禁止把不同语义层混写为同一类长期资产。

#### 场景:系统沉淀一次新内容
- **当** 系统准备将一次讲述沉淀为长期资产
- **那么** 原始录音与转写必须仍然保留为证据来源
- **那么** 客观事实、事件、叙事理解和表达特征必须能够在后续模块中分层处理
