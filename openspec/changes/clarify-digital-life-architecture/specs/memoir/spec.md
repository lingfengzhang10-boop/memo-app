## ADDED Requirements

### 需求:回忆录必须作为数字生命呈现面之一存在
回忆录必须被定义为数字生命资产的编排与阅读视图之一，而不是唯一的长期内容呈现方式。

#### 场景:用户查看回忆录
- **当** 用户打开回忆录
- **那么** 系统必须继续基于受治理的长期资产生成内容
- **那么** 系统必须允许回忆录与人生地图、关系图或传记视图并列存在

### 需求:回忆录必须继续保持来源可追溯
即使产品后续扩展更多呈现面，回忆录仍必须保持对底层资产的来源可追溯能力。

#### 场景:用户质疑某段回忆录内容
- **当** 用户要求解释某段回忆录来自哪里
- **那么** 系统必须能够追溯到相关事实、事件或源记忆
- **那么** 系统禁止将回忆录变成无法解释来源的自由生成结果
