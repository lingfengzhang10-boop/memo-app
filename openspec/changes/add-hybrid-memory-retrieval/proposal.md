## 为什么

当前分身对话主要依赖版本快照、facts / events 摘要和最近 transcript，无法形成“薪水联想到工资”“她联想到某个重要人物”“杭州那几年联想到一整段生活阶段”的真人式联想。要让分身更智能，必须把检索从静态快照升级为受治理的混合召回。

## 变更内容

- 新增混合记忆检索能力，统一协调语义、人物、地点、时间和稳定事实多路召回。
- 引入“记忆包”作为分身和其他呈现面的运行时上下文，而不是直接把整库数据塞给模型。
- 为语义召回补上可追溯的记忆切片与相似度检索路径。
- 为检索失败提供结构化降级路径，避免向量能力不可用时整个分身失忆。

## 功能 (Capabilities)

### 新增功能

- `hybrid-memory-retrieval`: 提供运行时多路召回、融合排序和记忆包生成能力

### 修改功能

- `twin`: 分身对话从“读取版本快照”升级为“读取受治理的记忆包”

## 影响

- 可能新增表：记忆切片、召回日志、向量索引相关结构
- 受影响 API：分身对话检索路径、可能新增检索服务端接口
- 受影响页面：分身对话页，以及后续任何需要实时记忆上下文的呈现面
- 受影响依赖：向量能力、Embedding 模型与检索排序逻辑
