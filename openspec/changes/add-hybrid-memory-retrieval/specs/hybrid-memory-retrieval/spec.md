## ADDED Requirements

### 需求:运行时记忆检索必须使用多路召回
系统必须在运行时同时协调语义、人物、地点、时间和稳定事实多路召回，而禁止仅依赖单一路由完成长期记忆检索。

#### 场景:用户提出模糊问题
- **当** 用户以同义词、代词、地点阶段或省略表达提出问题
- **那么** 系统必须同时尝试多路召回并汇总候选结果
- **那么** 系统禁止仅根据单条关键词匹配决定长期记忆命中

### 需求:系统必须生成可追溯的记忆包
系统必须把多路召回结果融合为一个可追溯的记忆包，供分身或其他消费面使用。

#### 场景:召回完成后准备交给消费面
- **当** 系统完成一次运行时记忆检索
- **那么** 输出必须包含高相关事实、事件以及相关人物或地点引用
- **那么** 输出必须保留来源记忆标识、命中路由信息和排序依据

### 需求:语义召回失败时必须有降级路径
如果语义向量路由不可用，系统必须降级到结构化召回与关键词召回，而禁止直接返回“无记忆可用”。

#### 场景:向量路由故障
- **当** Embedding 或向量检索能力临时不可用
- **那么** 系统必须继续通过已确认事实、事件和关键词匹配生成记忆包
- **那么** 系统必须允许消费面继续回答，但可降低联想能力
