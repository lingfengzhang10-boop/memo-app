## ADDED Requirements

### 需求:系统必须提供记忆工坊作为治理入口
系统必须提供一个专门的治理入口，用于查看、确认、修正、归档和追溯长期资产，而禁止把这些职责全部塞进聊天页或回忆录页。

#### 场景:用户想管理一条长期资产
- **当** 用户需要确认、纠正或归档某条长期资产
- **那么** 系统必须提供集中式的治理入口
- **那么** 系统必须允许用户查看该资产的来源和状态

### 需求:系统必须提供人生地图与关系图消费面
系统必须允许以人生阶段、地点和人物关系为入口阅读长期资产，而不局限于聊天或线性回忆录。

#### 场景:用户按地点或人物查看自己的人生内容
- **当** 用户切换到地点或人物视角查看长期资产
- **那么** 系统必须能够基于已确认事件、人物、地点和关系资产组织内容
- **那么** 系统禁止仅返回未经组织的 transcript 列表

### 需求:系统必须支持传记化输出
系统必须允许基于受治理资产生成适合阅读或导出的传记化输出，而不是要求用户手工从聊天记录整理。

#### 场景:用户想查看长期整理后的自我画像
- **当** 用户请求传记、纪念册或长期画像视图
- **那么** 系统必须能够从已确认且允许公开的资产中组织输出
- **那么** 系统必须保留对底层资产的来源追溯
