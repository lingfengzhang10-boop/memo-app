## 为什么

分身音色上传当前会把分身展示名直接作为上游 `customName` 传给克隆接口。分身名包含中文、空格或其他本地化字符时，上游会直接拒绝请求，导致一段有效样本也无法生成音色。

这是已经在真实使用中出现的阻塞问题。分身专属音色是当前分身体验的重要增强能力，不能要求用户为了通过上游约束而改掉自己的分身名字。

## 变更内容

- 在服务端为音色克隆请求生成符合上游约束的 `customName`，只允许 ASCII 字母、数字、`_` 和 `-`，并限制在 64 个字符以内。
- 当分身展示名不包含任何可保留字符时，服务端必须回退到安全的默认名字，而不是把上游校验错误直接暴露成失败根因。
- 保持分身展示名、页面文案和现有音色克隆返回结构不变，只修复上游请求边界。

## 功能 (Capabilities)

### 新增功能

无

### 修改功能

- `twin`: 分身专属音色克隆在遇到本地化分身名时，必须由服务端兜底生成可接受的上游名字，不能要求用户手动改名才能成功上传。

## 影响

- 受影响路由：`app/api/twin/voice/clone/route.ts`
- 受影响页面：`app/twin/[id]/page.tsx`（维持现有流程，仅消费修复后的结果）
- 无数据库变更
- 不影响记忆确认边界、一人一分身边界，也不影响画像层与客观记忆层的分离
