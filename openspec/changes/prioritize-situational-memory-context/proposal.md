## 为什么

当前分身在回答带有时间、地点或人物限定的问题时，仍然会优先命中全局 confirmed fact，而不是先收窄到该语境下的经历、压力和情绪线索。这会导致诸如“去杭州后怕什么”被回答成“怕突然死掉”，削弱“分身理解这段人生语境”的核心体验。

现在需要调整，是因为我们已经补上 semantic substrate，但运行时回答策略还停留在“全局标签优先”的阶段；如果不尽快修正，后续混合检索、人物地点图谱和叙事层都会继续被错误回答路径放大。

## 变更内容

- 调整分身回答编排：当用户问题包含明确的时间、地点、人物或阶段锚点时，先在局部语境内检索并组织答案，再回退到全局长期事实。
- 为记忆提取补充“担心 / 压力 / 最难 / 不稳定”等表达的中间语义，允许其进入 situational worry / stressor 类资产，而不是只能落到事件或客观状态。
- 收紧全局 fear fact 的回退条件，避免在局部语境证据已充分时仍直接输出全局恐惧结论。
- 增加运行时解释信息，能区分“局部语境回答”与“全局 fallback 回答”，便于调试和后续治理。

## 功能 (Capabilities)

### 新增功能
- `contextual-answer-routing`: 让分身在回答时先识别并绑定问题中的时间、地点、人物和阶段语境，再在该范围内组织答案。

### 修改功能
- `memory-companion`: 扩展对“担心 / 压力 / 最难 / 不稳定”等表达的提取语义，使其可进入可确认的局部情绪/压力资产，而不只是 facts / events。
- `twin`: 分身回答路径从“全局 confirmed fact 优先”调整为“局部语境优先，全局事实作为 fallback”，并要求在局部证据充分时避免被全局 fear fact 误导。

## 影响

- 受影响的代码：`app/api/twin/chat/route.ts`、语义检索与记忆包组装逻辑、facts / events / semantic annotation 相关服务。
- 受影响的 API：主页记忆提取、分身聊天运行时上下文组装。
- 受影响的数据：可能新增 situational worry / stressor 类结构化字段或候选资产，不改变“一人一分身”边界。
- 不改变的边界：
  - 记忆确认边界仍然成立，未确认语义不能直接成为 active twin truth。
  - 画像层仍不能被客观层编辑污染。
