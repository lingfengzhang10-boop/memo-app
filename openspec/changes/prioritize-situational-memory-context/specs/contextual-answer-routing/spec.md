## ADDED Requirements

### 需求:分身必须优先回答带锚点问题的局部语境
当用户问题包含明确的时间、地点、人物或阶段锚点时，系统必须先在该局部语境范围内组织答案，而不是直接以全局长期事实作答。

#### 场景:地点锚点优先于全局恐惧事实
- **当** 用户提问“去杭州后怕什么”这类带有地点和阶段锚点的问题
- **那么** 系统必须先检索与“杭州”“去杭州后”“2024”相关的 confirmed events、confirmed facts 和 semantic evidence
- **那么** 只有在局部语境不足以回答时，系统才可以回退到全局 fear fact

### 需求:系统必须显式记录回答路由
系统必须在运行时记下当前回答是基于局部语境还是全局 fallback，以及命中的主要锚点和回退原因。

#### 场景:局部语境回答被触发
- **当** 用户问题命中时间、地点、人物或阶段锚点
- **那么** 系统必须输出可追溯的 `answerMode`、`situationAnchors` 和主要证据来源

#### 场景:系统回退到全局事实
- **当** 局部语境证据不足以支持回答
- **那么** 系统必须记录 fallback 原因
- **那么** 不得让回退过程变成不可解释的 prompt 漂移
