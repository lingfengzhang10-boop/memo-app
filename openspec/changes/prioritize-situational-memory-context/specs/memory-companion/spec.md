## ADDED Requirements

### 需求:系统必须提取局部压力和担心候选
系统必须能够从“最难”“压力大”“担心”“不稳定”“悬着心”等表达中提取局部压力候选，用于后续确认和局部语境回答。

#### 场景:用户表达阶段性压力
- **当** 用户说“2024 年我搬到杭州后，最难的是收入不稳定”
- **那么** 系统必须至少产出一个与该阶段绑定的局部压力候选
- **那么** 该候选不得被强制提升为长期 `fear`

### 需求:局部压力候选必须遵守现有确认边界
局部压力和担心语义必须先作为候选线索呈现给用户确认，未确认前禁止进入正式长期资产层。

#### 场景:系统识别到担心语义
- **当** 系统从用户原话中识别出 `worry_about`、`stressor` 或 `situational_anxiety`
- **那么** 系统必须先以候选线索形式展示
- **那么** 未确认内容禁止写入 active twin truth

## MODIFIED Requirements

### 需求:Structured memory model
系统必须至少支持三类长期或半长期记忆：非事件记忆、时间轴事件，以及可确认的局部压力/担心语义。

#### 场景:Preference or fear is captured
- **当** 用户表达偏好、担心、边界、状态、位置等非事件信息
- **那么** 该信息必须进入 `memory_facts` 或对应的可确认候选资产

#### 场景:Life event is captured
- **当** 用户表达带有时间、年龄、阶段或经历转折的内容
- **那么** 该信息必须进入 `memory_events`

#### 场景:Situational worry is captured
- **当** 用户表达某一阶段里“最难”“最担心”“压力最大”“收入不稳定”这类局部压力
- **那么** 系统必须把它表示成与时间、地点或阶段绑定的候选语义
- **那么** 不得直接把它默认写成全局长期 `fear`
