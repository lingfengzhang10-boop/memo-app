## MODIFIED Requirements

### 需求: Speech-first daily conversation
系统必须继续以语音讲述作为默认入口，但在产品表面上必须把 Native App 视为高频的语音优先入口，把 Web 视为可保留的辅助记录与验证入口，而不是要求两端承载完全相同的交互密度。

#### 场景: 用户在 App 中自然讲述
- **当** 用户打开 Native App 并开始一段新的日常讲述
- **那么** 系统必须优先提供直接的语音录入能力
- **那么** 用户不得被要求先填写结构化表单或进入深层编辑页面

#### 场景: 用户在 Web 中继续验证或补充讲述
- **当** 用户通过 Web 使用记忆入口
- **那么** 系统必须允许其继续录音、查看转写、接收即时回应和完成确认
- **那么** Web 不必与 App 完全复制相同的系统能力交互，但必须保持相同的记忆真相

### 需求: Objective clues require user confirmation before persistence
从对话中抽出的客观线索必须继续在正式入库前经过用户确认，并且这条确认边界必须在 App 与 Web 上保持一致，不得因为客户端不同而改变正式记忆的成立条件。

#### 场景: 用户在 App 中确认线索
- **当** Native App 中出现 fact 或 event 候选线索
- **那么** 系统必须允许用户逐条确认、关闭或稍后处理
- **那么** 只有被确认的线索才能进入正式记忆层

#### 场景: 用户在 Web 中确认线索
- **当** Web 中出现 fact 或 event 候选线索或待整理项
- **那么** 系统必须遵守与 App 相同的确认规则
- **那么** 未确认线索不得因为 Web 是深度工作台而被直接视为正式记忆
