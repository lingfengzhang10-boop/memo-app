## ADDED Requirements

### 需求:记忆使用策略必须位于召回和生成之间
系统必须在检索召回之后、模型生成之前构造记忆使用策略，决定哪些记忆允许进入本轮回答以及本轮应该采用什么回答推进模式。

#### 场景:召回结果进入策略层
- **当** 分身聊天链路获得 semantic chunks、confirmed facts、confirmed events 或 topic interactions
- **那么** 系统必须先构造本轮 `MemoryUsePolicy`
- **那么** 系统必须输出本轮允许使用的记忆包
- **那么** 系统禁止把未经裁剪的召回结果直接整体交给模型生成回答

#### 场景:策略层保留调试解释
- **当** 某条候选记忆被允许或阻断
- **那么** 系统必须在服务端可观测信息中保留原因
- **那么** 系统禁止把 `answerProgressionMode`、`blockedMemoryReasons` 等内部字段原样作为用户可见回复

### 需求:记忆必须具备回答准入状态
系统必须区分原始记录、候选记忆、可信记忆、长期稳定记忆和只留档内容，且回答阶段必须优先使用可信或长期稳定内容。

#### 场景:原始输入只作为证据
- **当** 用户输入只存在于原始 transcript 或 raw chunk 中
- **那么** 系统可以保留其作为证据来源
- **那么** 系统禁止将其默认视为 confirmed fact 或 stable memory

#### 场景:低置信输入不主动参与分身回答
- **当** 输入被判定为玩笑、试探、误收音、ASR 异常或上下文不完整
- **那么** 系统必须将其标记为低准入或只留档
- **那么** 系统禁止在分身回答中主动把该内容当作真实经历使用

#### 场景:已确认记忆可以参与回答
- **当** 记忆已经进入 confirmed facts 或 confirmed events
- **那么** 策略层可以将其纳入本轮可用记忆包
- **那么** 策略层仍必须根据当前 topic scope 判断它是否属于本轮问题

### 需求:记忆包必须受当前话题边界约束
系统必须根据用户问题中的时间、地点、人物、阶段和话题意图构造当前 topic scope，并只允许范围内的可信记忆进入本轮回答。

#### 场景:杭州话题只能使用杭州相关记忆
- **当** 用户询问“你在杭州过得怎么样”或“还有别的吗”
- **那么** 策略层必须优先使用与杭州阶段相关的可信事实、事件和证据
- **那么** 策略层禁止为了补充答案而引入无关地点、无关人物或无关人生阶段的记忆

#### 场景:否定式试探不能诱导编造
- **当** 用户问“你在杭州有没有朋友聚会”且当前可用记忆包中没有该事件
- **那么** 分身必须表达不确定或没有明确记得
- **那么** 系统禁止让模型编造朋友聚会、吃饭、玩乐等未确认事件

### 需求:回答推进模式必须控制重复和收束
系统必须为同一 topic 的连续对话选择回答推进模式，使分身能够首次回答、深化、换角度、模糊唤回或自然收束。

#### 场景:短时间重复同一问题
- **当** 同一 asker 在短时间内重复询问同一 topic
- **那么** 策略层必须选择 `deepen_answer`、`diversify_answer` 或 `graceful_close`
- **那么** 分身禁止原样复述上一轮主句

#### 场景:用户追问还有别的吗
- **当** 用户围绕同一 topic 追问“还有别的吗”
- **那么** 策略层必须先寻找当前 topic 下未讲过的可信细节
- **那么** 如果没有新增可信细节，分身必须自然收束
- **那么** 分身禁止跨 topic 抓取其他记忆填补空白

#### 场景:几小时或几天后重问
- **当** 同一 asker 在较长时间后再次询问已聊过 topic
- **那么** 策略层可以选择 `fuzzy_recall`
- **那么** 分身可以自然表达“好像之前聊到过这段”
- **那么** 分身禁止精确机械地暴露内部时间记录

### 需求:程序性记忆必须独立于人生事实
系统必须把“怎么回答”的规则作为程序性记忆或策略配置处理，禁止将其写入人生事实、事件或 active twin truth。

#### 场景:程序性规则约束表达
- **当** 策略层判断本轮应避免重复或自然收束
- **那么** 系统必须把该规则作为表达策略提供给模型
- **那么** 系统禁止把该规则写入 `memory_facts`、`memory_events` 或 `twin_versions`

#### 场景:程序性规则失败时可回滚
- **当** 策略层导致回答质量明显下降
- **那么** 系统必须能够通过配置或代码路径临时关闭策略层
- **那么** 系统必须保留现有 situational routing 作为回退路径

### 需求:固定 QA 必须覆盖记忆使用风险
系统必须提供可重复执行的 QA 验证方式，覆盖重复回答、跨主题污染、未确认事件编造和自然收束。

#### 场景:杭州记忆回归测试
- **当** 测试用例依次发送“你在杭州过得怎么样”“你在杭州过得怎么样”“还有别的吗”
- **那么** 第二轮回答必须避免原样重复第一轮
- **那么** 第三轮回答必须只使用杭州 topic 下可信且未讲过的内容，或自然收束

#### 场景:未确认事件防幻觉测试
- **当** 测试用例询问数据库中不存在的朋友聚会、辣条或其他未确认事件
- **那么** 分身必须表达不确定或未明确记得
- **那么** 分身禁止把其他记忆套入当前问题
