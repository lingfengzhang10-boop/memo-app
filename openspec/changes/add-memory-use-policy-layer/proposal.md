## 为什么

当前记忆系统已经具备原始证据、结构化事实/事件、语义召回、语境收窄和话题互动记忆，但分身回答仍容易把“检索到的内容”直接当作“本轮应该说的内容”。这会导致同一话题反复复述、追问“还有别的吗”时跨主题硬套记忆、以及把不稳定输入误当成长期事实。

这次变更的目标是补齐“记忆使用策略层”：让系统先判断一条记忆是否可信、是否属于当前问题、是否应该在本轮说、应该说到什么程度，再把裁剪后的记忆包交给模型生成自然回答。

## 变更内容

- 新增记忆使用策略层，统一处理记忆准入、记忆包裁剪、回复推进模式和程序性回答规则。
- 为日常输入增加更明确的记忆准入判定，区分原始记录、候选记忆、可信记忆、长期稳定记忆和只留档不参与回答的内容。
- 调整分身对话链路，使检索召回结果必须先经过“本轮可用记忆包”裁剪，再进入模型 prompt。
- 为同一话题短期重复、追问“还有别的吗”、几小时/几天后重问等场景定义回答推进策略。
- 引入程序性记忆规则，约束“怎么说”，例如避免原样复述、禁止编造未确认事件、没有新细节时自然收束。
- 增加固定 QA 回归用例，覆盖杭州记忆、重复提问、追问、否定式试探和未确认事件防幻觉。

## 功能 (Capabilities)

### 新增功能

- `memory-use-policy`: 定义记忆从“被存储/被召回”到“本轮允许被使用”的策略层，包括准入状态、记忆包裁剪、回答推进模式、程序性记忆和验证要求。

### 修改功能

- `twin`: 分身对话必须通过记忆使用策略层组织回答，不能直接把检索结果或全部相关摘要交给模型自由发挥。
- `memory-companion`: 日常记忆沉淀必须为输入内容标记可信度与准入状态，避免玩笑、误收音、未确认或低置信内容直接影响分身回答。

## 影响

- 受影响 API：`app/api/twin/chat/route.ts`、日常记忆提取/写入相关 API、可能的调试输出。
- 受影响服务层：语义记忆召回、situational routing、topic interaction memory、prompt/context packing。
- 受影响数据库：优先复用现有表与字段；如现有字段不足，可为记忆 chunk / fact / event 增加轻量治理字段或新增最小策略辅助表。
- 受影响边界：不改变“一人一分身”；不允许未确认内容直接进入正式 facts/events 或 active twin truth；画像层仍不得被客观记忆编辑直接污染。
