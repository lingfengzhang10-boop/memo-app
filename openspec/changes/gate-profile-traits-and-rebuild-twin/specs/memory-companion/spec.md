## 修改需求

### 需求:长期画像提取必须经过候选和过滤阶段
系统必须将画像提取从“提取后直接 merge”调整为“提取候选 -> 累计/过滤 -> 再进入长期画像投影”。系统禁止把单条 transcript 的画像增量直接写成长期人格资产。

#### 场景:一次录音结束后触发画像提取
- **当** 首页或其他日常录音链路完成 transcript 并触发 profile extraction
- **那么** 系统必须先产出画像候选 trait
- **那么** 系统禁止直接把该次提取结果整体 merge 进长期画像

#### 场景:同一 trait 多次出现后进入长期画像
- **当** 多条 transcript 对同一画像 trait 提供支持，且达到准入门槛
- **那么** 系统必须基于 vetted traits 重建长期画像投影
- **那么** 长期画像中对应字段必须来自 vetted traits，而不是任意单次提取

### 需求:长期画像投影不得绕过事实和事件确认边界
系统必须保持客观记忆层与画像层分离。facts / events 的确认边界不变，画像投影不得把未确认的客观内容直接升级为长期真相。

#### 场景:提取结果包含疑似客观事实
- **当** 画像提取结果中出现可被解释为客观事实的内容
- **那么** 系统必须优先将其留在候选层或交由 facts/events 路径处理
- **那么** 系统禁止仅凭这次画像提取就把它作为长期 lifeFact 固化
