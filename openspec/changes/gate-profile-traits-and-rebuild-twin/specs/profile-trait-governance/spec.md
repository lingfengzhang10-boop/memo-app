## 新增需求

### 需求:画像 trait 必须经过准入治理后才能进入长期画像
系统必须将画像 trait 区分为候选 trait 与 vetted trait。单条 transcript 提取出的画像内容禁止直接进入长期画像或分身当前真相。

#### 场景:单条 transcript 提取到新的画像候选
- **当** 系统从一次 transcript 中提取出新的 `lifeFact`、`lexicalHabit`、`memoryTheme` 或 `twinNote` 候选
- **那么** 系统必须先将其记录为候选 trait
- **那么** 系统禁止直接把该候选写入长期画像投影

#### 场景:候选 trait 达到准入门槛
- **当** 某个候选 trait 达到支持次数、信任分数和最近出现时间的准入门槛
- **那么** 系统必须将其状态提升为 vetted
- **那么** 只有 vetted trait 才能进入长期画像投影

### 需求:画像 trait 必须保留来源和信任信息
系统必须为每个画像 trait 记录来源、支持次数、信任分数和最近出现时间，以便后续清洗、降权和重建。

#### 场景:系统累积同一 trait 的支持证据
- **当** 新 transcript 再次支持已存在的候选 trait
- **那么** 系统必须增加该 trait 的支持次数
- **那么** 系统必须更新其来源 memory 和最近出现时间

#### 场景:低信任输入命中既有 trait
- **当** transcript 的输入信任等级较低，或命中噪音/环境音/玩笑话标签
- **那么** 系统必须降低该次输入对 trait trust score 的贡献
- **那么** 系统禁止因为单次低信任输入将候选 trait 直接提升为 vetted

### 需求:系统必须支持对画像 trait 降权和剔除
系统必须支持将不再可信、长期未再出现或被证伪的画像 trait 标记为 rejected 或 stale，并从长期画像投影中移除。

#### 场景:明显噪音 trait 被识别
- **当** 某个画像 trait 被判定为明显噪音、ASR 补全或环境音产物
- **那么** 系统必须允许将其标记为 rejected
- **那么** 该 trait 禁止继续出现在长期画像投影和分身版本构建中

#### 场景:长期未再出现的 trait
- **当** 某个 vetted trait 长期未再被支持且缺乏其他正式资产支撑
- **那么** 系统必须允许将其降为 stale
- **那么** stale trait 默认不应作为高优先级分身画像输入
