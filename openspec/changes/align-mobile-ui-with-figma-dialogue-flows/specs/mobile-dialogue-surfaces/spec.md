## ADDED Requirements

### 需求:移动端主入口必须以统一对话页承载文字与语音两种输入形态
系统必须将移动端主入口收束到同一套对话页面中，并以文字模式作为默认展示形态；用户必须可以在不离开当前人物和当前对话上下文的情况下切换到语音模式。

#### 场景:进入移动端默认看到文字对话页
- **当** 用户进入移动端主入口且未显式指定输入模式
- **那么** 系统必须展示当前人物的顶部回复文本、中部人物展示区域、底部文字输入框
- **那么** 系统必须在输入区左侧提供切换到语音模式的入口
- **那么** 系统必须在输入区右侧保留附件加号入口

#### 场景:从文字模式切换到语音模式
- **当** 用户在统一对话页点击语音模式切换入口
- **那么** 系统必须在同一条对话链路里切换为“按住说话”的语音操作区
- **那么** 当前人物、当前已展示的回复文本和当前对话上下文不得丢失
- **那么** 系统必须同时提供返回文字模式的入口

### 需求:统一对话页必须同时回显人物回复文本与用户话语文本
系统必须在统一对话页里持续显示当前人物最近一次的回复文本，并把用户通过文字输入或语音输入提交的话语以文字形式回显在页面底部。

#### 场景:用户提交文字消息
- **当** 用户在文字模式下发送文本
- **那么** 系统必须在页面底部展示该条用户文本
- **那么** 系统必须在顶部人物回复区展示随后生成的人物回复文本

#### 场景:用户提交语音消息
- **当** 用户在语音模式下完成一段语音输入且系统得到可用 transcript
- **那么** 系统必须将该段用户话语以文字形式展示在页面底部
- **那么** 系统必须在顶部人物回复区展示随后生成的人物回复文本

### 需求:人物选择页必须显示可对话人物并回跳到统一对话页
系统必须提供单独的人物选择页，显示系统默认人物、当前用户自己的分身，以及显式授权给当前用户的其他可对话人物；用户选择自己的分身后，统一对话页必须切换到分身对话链路，而不是回到 Liva。

#### 场景:打开人物选择页
- **当** 用户点击统一对话页顶部的人物头像或名字区域
- **那么** 系统必须进入人物选择页
- **那么** 人物选择页必须显示系统默认人物 Liva
- **那么** 人物选择页必须显示当前用户自己的分身（如果已创建）
- **那么** 人物选择页必须显示已授权给当前用户的其他人物

#### 场景:切换当前人物
- **当** 用户在人物选择页点击某个可对话人物卡片
- **那么** 系统必须以蓝色光晕或等价的激活态标识当前被选中的人物
- **那么** 系统必须切换当前对话人物并返回统一对话页
- **那么** 如果用户选择的是自己的分身，后续消息必须走 `/api/twin/chat` 分身链路

### 需求:授权人物必须展示创建分身时上传的独立照片素材
系统必须为每个可授权对话的分身展示其创建时上传的独立照片素材；对于已授权人物，系统禁止使用统一占位头像替代真实照片素材。

#### 场景:展示已授权人物卡片
- **当** 用户在人物选择页看到某个已授权人物
- **那么** 系统必须展示该人物分身创建时上传的独立照片素材
- **那么** 系统不得使用共享占位头像替代该素材

#### 场景:展示分身详情
- **当** 用户在“更多”页进入自己的分身详情
- **那么** 系统必须展示该分身创建时上传的独立照片素材
- **那么** 该素材必须与后续授权人物卡片读取的素材来源保持一致

### 需求:移动端必须以“更多”页承接二级入口并保留预留态
系统必须通过“更多”页承接分身入口、用户信息入口和 Figma 中未细化的其他操作；对于尚未实现的功能，系统必须显示明确预留态而不是消失或静默失败。

#### 场景:用户尚未创建自己的分身
- **当** 用户从统一对话页进入“更多”页且当前用户还没有自己的分身
- **那么** 系统必须展示“未创建分身”版本的更多页
- **那么** 系统必须提供进入现有 bootstrap 问答页的创建入口
- **那么** 系统必须提供进入用户信息页的齿轮入口

#### 场景:用户已经创建自己的分身
- **当** 用户从统一对话页进入“更多”页且当前用户已经有自己的分身
- **那么** 系统必须展示分身卡片并允许进入分身详情页
- **那么** 分身详情页必须提供“与分身对话”的主操作
- **那么** 其他暂未细化的操作必须显示为预留态

#### 场景:用户点击附件加号或其他预留操作
- **当** 用户点击附件加号、本地图片或视频入口，或其他尚未实现的操作位
- **那么** 系统必须展示明确的预留态反馈
- **那么** 系统不得伪装成已经完成上传、保存或跳转成功
