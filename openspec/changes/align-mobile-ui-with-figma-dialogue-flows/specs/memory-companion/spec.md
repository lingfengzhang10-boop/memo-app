## MODIFIED Requirements

### 需求:Speech-first daily conversation
系统必须在统一的日常对话界面中同时支持文字与语音两种输入形态，并保持“先对话、后结构化”的主链路；移动端进入时默认展示文字对话页，但用户必须可以在同一页面内一键切换到语音输入，而不是进入表单式流程。

#### 场景:用户进入日常陪伴对话页
- **当** 用户打开移动端日常陪伴入口
- **那么** 系统必须先展示统一对话页而不是结构化表单或工具页
- **那么** 系统必须默认提供文字输入形态
- **那么** 系统必须允许用户随时切换到语音输入形态

#### 场景:用户通过统一对话页继续讲述
- **当** 用户在统一对话页中通过文字或语音继续讲述
- **那么** 系统必须继续沿用现有的即时回复、线索提取与确认链路
- **那么** 系统不得因为输入形态变更而改变“先确认再入库”的边界
