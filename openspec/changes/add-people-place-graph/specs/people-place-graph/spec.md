## ADDED Requirements

### 需求:系统必须把人物与地点作为长期资产处理
系统必须支持将已确认的人物与地点沉淀为长期资产，而禁止仅把它们留在零散 transcript 或描述字段中。

#### 场景:用户多次提到同一个人物或地点
- **当** 用户在不同讲述中反复提到同一个人物或地点
- **那么** 系统必须能够把这些提及归并到同一长期实体
- **那么** 系统必须保留实体与来源记忆之间的链接

### 需求:人物与地点候选必须经过确认
人物与地点候选在首次抽取后必须经过用户确认或纠正，才能进入正式长期资产层。

#### 场景:后台抽取到新的人物或地点
- **当** 系统从一次讲述中抽取到人物或地点候选
- **那么** 系统必须先将其展示为待确认内容
- **那么** 系统禁止直接把未经确认的候选写入正式人物或地点资产

### 需求:人物与地点必须支持规范名与别名
系统必须允许每个人物和地点同时保存规范名和多个别名，以支持真实口语中的称呼差异。

#### 场景:同一对象出现多种叫法
- **当** 用户使用不同叫法提到同一人物或地点
- **那么** 系统必须允许这些叫法被归并为同一实体
- **那么** 系统必须保留原始叫法以支持后续检索和解释
