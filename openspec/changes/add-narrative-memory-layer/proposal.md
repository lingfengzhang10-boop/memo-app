## 为什么

现在系统已经能记住“发生了什么”，但还不能稳定记住“用户怎么理解这件事”。如果没有叙事层，分身只能回答事实，无法回答“为什么这件事对我重要”“我后来为什么改口”“我反复绕不过去的主题是什么”。

## 变更内容

- 新增叙事记忆层，把用户对经历的解释、重述和意义提炼保存为独立资产。
- 允许同一事实或事件对应多个叙事版本，而不是被最新说法简单覆盖。
- 为 turning points、recurring patterns、life themes、unresolved knots 等意义标签建立长期沉淀路径。
- 让分身、回忆录和未来传记能够在事实之外输出“这个人怎样理解自己”。

## 功能 (Capabilities)

### 新增功能

- `narrative-memory`: 提供叙事记忆、重述版本和意义标签能力

### 修改功能

- `memory-companion`: 从抽取客观信息升级为也能沉淀主观叙事理解
- `twin`: 分身能够在“为什么”类提问中使用叙事层，而不是只复读事实
- `memoir`: 回忆录允许在事实时间线之上叠加叙事理解和人生主题

## 影响

- 可能新增表：叙事记忆、意义标签、叙事与事实/事件关联
- 受影响 API：后台抽取、确认流、分身对话上下文构建、回忆录编排
- 受影响页面：未来记忆工坊、回忆录、传记页、分身对话页
