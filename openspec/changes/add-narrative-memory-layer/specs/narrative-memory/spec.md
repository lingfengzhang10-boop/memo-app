## ADDED Requirements

### 需求:系统必须把叙事理解作为独立长期资产
系统必须允许用户对经历的解释、重述和理解以独立长期资产存在，而禁止把这些主观内容直接并入客观事实层。

#### 场景:用户解释一段经历的意义
- **当** 用户不仅描述经历，还说明这件事为什么重要或后来怎样理解它
- **那么** 系统必须允许把这段解释沉淀为独立叙事资产
- **那么** 系统禁止直接把该解释改写为客观事实

### 需求:叙事资产必须支持多版本并存
系统必须允许同一主题的多种叙事理解并存，以反映用户前后理解的变化。

#### 场景:用户对同一经历前后改口
- **当** 用户在不同时间对同一经历给出不同解释
- **那么** 系统必须允许保留多个叙事版本
- **那么** 系统必须保留每个版本的来源和时间信息

### 需求:系统必须支持意义标签的保守提炼
系统必须能够为叙事资产提炼转折点、反复模式、人生主题或未解心结等意义标签，但在不确定时必须保守降级。

#### 场景:AI 无法稳定提炼意义标签
- **当** 系统无法可靠地为某条叙事提炼意义标签
- **那么** 系统必须仍可保存叙事文本本身
- **那么** 系统禁止为了凑齐结构而编造意义标签
