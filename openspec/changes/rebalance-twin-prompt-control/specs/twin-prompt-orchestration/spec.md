## ADDED Requirements

### 需求:分身回答必须使用收敛后的 prompt 结构
系统必须将分身回答 prompt 组织为“真相边界 + 当前语境包 + 表达提示”三层，而不是无限叠加显式控制规则。

#### 场景:构建分身回答 prompt
- **当** 系统为分身生成某一轮回答 prompt
- **那么** prompt 必须包含少量不可违背的真相边界
- **那么** prompt 必须包含与当前问题最相关的语境包
- **那么** prompt 必须包含少量关于自然表达的提示
- **那么** 系统不得把大量重复的系统控制条款直接堆叠进 prompt

### 需求:分身回答必须使用轻量 few-shot 覆盖高频对话推进场景
系统必须使用少量 few-shot 或等效示例来覆盖高频的对话推进场景，以提升回答自然度。

#### 场景:处理同一话题的追问
- **当** 当前问题属于“刚聊过再问”“同日重问”“几天后再问”或“没有新角度时收束”这类高频场景
- **那么** 系统必须通过少量示例指导模型如何自然推进
- **那么** 系统不得仅依赖新增规则来硬性规定句子应该怎么写

### 需求:系统术语不得原样暴露给模型
系统必须把运行时路由状态翻译为自然语言提示后再提供给模型，禁止把内部术语直接作为 prompt 核心内容。

#### 场景:存在 progression 或 recency 信号
- **当** 系统已经得出 progression、recency 或 fallback 相关信号
- **那么** 系统必须将这些信号翻译为自然语言约束或示例
- **那么** 系统不得要求模型直接围绕 `answerProgressionMode`、`topicRecencyBand`、`fallbackReason` 等术语作答
