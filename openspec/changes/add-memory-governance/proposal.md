## 为什么

记忆越多，系统越需要知道哪些可以大胆说、哪些只能保守说、哪些已经过时、哪些只是候选。没有统一的记忆治理，分身即使更聪明，也会在敏感内容暴露、旧记忆误用、修正覆盖和矛盾内容处理上失控。

## 变更内容

- 新增记忆治理能力，统一定义生命周期、可见性、敏感性、版本与证据链。
- 让各类长期资产都能区分 candidate / confirmed / stable / stale / archived。
- 让系统保存修正、重述、改口和冲突记忆，而不是简单覆盖。
- 让分身、回忆录和未来呈现面都遵守统一的治理规则。

## 功能 (Capabilities)

### 新增功能

- `memory-governance`: 提供生命周期、可见性、版本和证据链治理能力

### 修改功能

- `memory-companion`: 长期资产沉淀从“抽取并入库”升级为“抽取、确认、治理、演化”
- `twin`: 分身运行时必须遵守记忆可见性与敏感性约束
- `memoir`: 回忆录和未来输出面必须遵守记忆状态与来源规则

## 影响

- 受影响表：长期资产状态字段、可见性字段、版本链字段、来源链字段
- 受影响 API：确认流、编辑流、分身运行时过滤逻辑
- 受影响页面：记忆确认、回忆录编辑、未来记忆工坊和传记输出
