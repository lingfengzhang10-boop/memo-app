## 1. 治理模型

- [ ] 1.1 定义长期资产的生命周期状态与状态转移规则
- [ ] 1.2 定义可见性和敏感性规则，以及它们对各消费面的约束
- [ ] 1.3 定义修正、改口、重述和归档的版本链规则

## 2. 链路接入

- [ ] 2.1 将确认流接入生命周期状态，而不是只区分“入库/未入库”
- [ ] 2.2 将编辑流接入 supersede 或版本链，而不是直接覆盖旧内容
- [ ] 2.3 将分身和回忆录读取逻辑接入治理过滤规则

## 3. 验证

- [ ] 3.1 验证敏感记忆不会被分身主动暴露
- [ ] 3.2 验证旧记忆变 stale 或 archived 后不会污染日常回答
- [ ] 3.3 验证修正后仍能追溯原始来源和历史版本
