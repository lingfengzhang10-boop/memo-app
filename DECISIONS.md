# 念及关键决策

## 1. 产品主入口保持语音优先
- 念及的主入口不是表单，不是时间线编辑器，而是语音讲述。
- 用户先讲，系统先接住，再逐步结构化整理。
- Web 当前作为优先入口，后续如需提升录音稳定性与自动播放体验，再补 App。

## 2. 客观记忆与画像分层存储
- `memories` 保存原始录音、转写与处理状态。
- `memory_facts` 保存非事件客观记忆，例如偏好、担忧、边界、位置、状态。
- `memory_events` 保存时间轴事件。
- `companion_profiles` 保存聚合画像，例如风格、口头禅、叙事习惯、情绪标记。
- 客观事实和画像不能混写，避免用户修正事实时污染性格层。

## 3. facts / events 先确认再入库
- 从用户讲话中抽取出的客观线索，不直接写入长期记忆。
- 先以气泡形式展示给用户逐条确认。
- 用户可确认、关闭或语音纠正。
- 只有用户确认后的 facts / events 才进入正式记忆层。

## 4. 回忆录是编排层，不是原始真相层
- 回忆录基于 `memory_events` 与 `memory_facts` 整理生成。
- 用户可以查看与编辑回忆录。
- 回忆录编辑回写客观事实与事件层，不回写画像层。
- 回忆录不替代原始音频与 transcript，它只是更适合阅读的编排结果。

## 5. 分身采用“快速冷启动 + 持续成长”
- 终极形态是用户长期讲述后形成更完整的数字分身。
- 为了降低冷启动无聊感，先做 12 题语音建模，快速生成初版分身。
- 冷启动分身不是一次性产品，而是分身成长链路的起点。
- 后续日常对话继续补全该分身的记忆、表达和人格。

## 6. 一人只允许一个分身
- 每个用户只允许拥有一个分身。
- 不支持“张三 A / 张三 B”式多分身。
- 该分身代表用户本人，只允许持续丰富，不重复新建。
- 代码层已经按单分身逻辑处理；数据库层需继续同步 `twin_profiles.user_id` 唯一约束。

## 7. 分身层与用户原始画像层分离
- `twin_profiles` 是分身主档，不等于 `companion_profiles`。
- `twin_versions` 保存可回溯的分身快照。
- `twin_bootstrap_sessions` 与 `twin_bootstrap_answers` 保存冷启动建模过程。
- 用户原始记忆层先沉淀，再汇总成分身层，不直接拿分身表充当真相源。

## 8. 分身对话先用版本快照，再逐步引入动态记忆
- 当前分身对话优先读取 `active_version_id` 对应版本快照。
- 同时叠加 `factsSnapshot`、`eventsSnapshot`、人格摘要与边界规则。
- 已加入表达层读取，不再只靠一句 seed prompt 做角色扮演。
- 后续继续增强：动态读取更多已确认记忆与长期表达样本库。

## 9. 表达层单独建模，不与记忆层混为一谈
- “它记得什么”与“它怎么说话”是两层能力。
- 冷启动 12 题会生成一版表达快照。
- 分身对话时，还会叠加最近日常对话中的真实表达痕迹。
- 后续方向是建立分身长期表达样本库，而不是只靠 prompt 模拟口气。

## 10. 分身语音优先走专属参考音色
- 浏览器本机 TTS 只作为兜底，不作为最终拟人方案。
- 当前已支持在分身对话页录制一段样本音频，生成 `voice uri`。
- 分身回复时优先尝试用这条专属音色合成语音，同时保留文字。
- 该能力依赖第三方语音供应商能力和账户权限，若供应商拒绝，应明确报错而不是静默失败。

## 11. 错误率的主要风险在 ASR 与音频质量，不在页面本身
- Web 会影响录音质量上限，但不会单独决定语义理解质量。
- 口语、省略句、方言、专有名词错误主要来自 ASR 和后续抽取放大。
- 因此系统设计重点不是“避免任何错误”，而是“避免错误直接污染长期记忆”。
- 当前通过确认、纠正、编辑与删除机制降低污染风险。

## 12. 对外授权是后续能力，不直接默认等同本人
- 分身未来可以授权给其他用户对话。
- 但外部用户带来的新线索不能直接写入正式记忆。
- 这些外部线索未来应带来源与待确认状态，再决定是否并入用户正式记忆。
- 产品文案上不承诺“完全等同本人”，而强调“基于用户讲述持续成长的数字分身”。

## 13. 当前遗留表 `ai_scripts` 不属于主链路
- `ai_scripts` 是早期遗留表。
- 当前代码与最新 schema 不再依赖该表。
- 它可以暂时保留，但不属于当前正式产品结构。
- 后续如做数据库清理，可统一删除遗留结构。
